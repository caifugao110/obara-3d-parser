# 故障排查

本文汇总运行、网格化、求解和打包时的常见问题。

## 程序无法启动

### 现象

运行 `python run.py` 后直接退出或出现 Qt/VTK 错误。

### 处理

1. 确认 Python 版本：

```powershell
python --version
```

2. 重新安装依赖：

```powershell
pip install -e ".[dev]" --upgrade
```

3. 检查 OpenGL/显卡驱动。
4. 运行测试确认核心模块：

```powershell
pytest
```

5. 如果提示找不到材质库，确认 `material_data/sldmaterials.json` 文件存在。

## 导入 STEP 失败

### 可能原因

- 文件不是有效 STEP 实体模型。
- 文件路径包含特殊权限或被占用。
- Gmsh/OpenCASCADE 无法解析该模型。
- 模型几何存在严重错误。

### 处理

- 用 CAD 软件重新导出 STEP。
- 尝试简化模型后导入。
- 将文件复制到不含特殊字符的短路径下再试。
- 先使用内置测试梁确认应用本身正常。

## 网格化失败

### 可能原因

- 网格尺寸过小或过大。
- 几何存在狭缝、重叠面、自交或非流形结构。
- 多实体模型边界复杂。

### 处理

- 先增大网格尺寸。
- 在 CAD 软件中修复几何。
- 对复杂模型分步简化。
- 确认模型单位与预期一致。

## 无法运行仿真

运行前必须满足：

- 已成功生成实体网格。
- 所有零件已分配材料（多零件模型时，每个零件都需要分配材料）。
- 至少存在一个固定面。
- 至少存在一个载荷。

如果缺少条件，程序会在弹窗中列出缺失项，包括哪些零件缺少材料。

## 求解失败或矩阵奇异

### 可能原因

- 约束不足，模型发生刚体运动。
- 约束面与受载面选择错误。
- 网格存在退化单元。
- 材料参数异常，例如弹性模量为 0。

### 处理

- 增加合理固定约束。
- 检查载荷方向和大小。
- 重新网格化并调整网格尺寸。
- 更换或修正材料属性。

## 位移或应力结果不合理

### 检查清单

- 模型尺寸单位是否按 mm 输入。
- 压力/力单位是否符合预期。
- 载荷方向是否正确。
- 固定约束是否过强或过弱。
- 网格是否过粗。
- 材料屈服强度和弹性模量是否正确。

建议先用测试梁复现实验流程，再分析复杂模型。

## 拾取不到目标面

### 处理

- 旋转视图，使目标面清晰可见。
- 放大目标区域。
- 切换网格显示，确认表面三角片分布。
- 如果模型面被拆分，可能需要选择多个面或简化几何。

## CalculiX 找不到或运行失败

检查：

```powershell
where ccx
ccx -v
```

如果未加入 `PATH`，设置环境变量：

```powershell
$env:OBARA_CALCULIX_CCX = "C:\path\to\ccx.exe"
```

或：

```powershell
$env:CCX_PATH = "C:\path\to\ccx.exe"
```

如果 `ccx` 能启动但求解失败，检查导出的 `.inp` 文件和 CalculiX 输出日志。

## 打包版启动失败

### 检查清单

- 是否分发了整个 `dist/obara-3d-parser/` 目录。
- 目录中是否存在 `material_data/` 和 `assets/`。
- 是否缺少 Gmsh、Qt、VTK 或 SciPy DLL。
- 杀毒软件是否隔离了部分文件。
- 目标机器显卡驱动是否支持 OpenGL。

可在开发机重新运行：

```powershell
.\build.ps1
```

并查看构建日志中的 PyInstaller 警告。

## PowerShell 无法运行脚本

如果出现执行策略错误：

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\build.ps1
```

该命令只影响当前 PowerShell 会话。

## 材料库加载失败

### 现象

加载外部材料库文件夹后，材料库面板为空或提示"未找到有效的材质数据文件"。

### 可能原因

- 文件夹中没有 `*.json` 文件。
- JSON 文件格式不正确，缺少必要字段。
- JSON 文件编码不是 UTF-8。

### 处理

1. 确认文件夹中存在至少一个 `*.json` 文件。
2. 检查 JSON 文件格式是否符合要求（见 [materials.md](materials.md)）。
3. 确保 JSON 文件使用 UTF-8 编码。
4. 检查 JSON 文件是否包含 `classifications` 数组和必要的材料属性。

## 切换材料库后当前材料丢失

### 现象

加载新的材料库后，当前分配的材料被移除并提示重新选择。

### 原因

当前使用的材料在新的材料库中不存在。

### 处理

1. 在新的材料库中选择替代材料。
2. 如果需要保留原有材料，将其添加到新的材料库 JSON 文件中。

## 多零件模型网格化失败

### 现象

多零件模型网格化时出现错误或生成的网格不完整。

### 可能原因

- STEP 文件中的零件之间存在间隙或干涉。
- 零件几何过于复杂或退化。
- 网格尺寸设置不合理。

### 处理

1. 增大网格尺寸后重试。
2. 在 CAD 软件中检查并修复零件间的几何关系。
3. 尝试将复杂模型拆分为单个零件分别分析。

## 多零件模型颜色标记不显示

### 现象

应用零件颜色后，视图中没有变化。

### 可能原因

- 网格尚未生成（零件颜色需要四面体网格支持）。
- 选中的零件索引不正确。

### 处理

1. 确保已完成网格化。
2. 在多零件模型中通过点击 3D 视图确认选中了正确的零件。
3. 尝试使用"重置布局"功能恢复默认视图。
